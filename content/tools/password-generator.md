# 密码生成器 — 强密码、随机密码、自定义规则

## 这是什么?

密码生成器是一种自动产生随机字符串的工具,产生的密码人手难以构造。最强的密码在所选字符集内均匀分布随机,具有足够的熵抵抗离线暴力破解(重要账户 ≥ 80 bits),并避免常见模式(生日、`qwerty`、`password123`)。本工具使用浏览器内置的 `crypto.getRandomValues()` API — 与 1Password、Bitwarden 等专业密码管理器使用同一熵源。

## 怎么用?(3 步)

1. **选择模式**:在工具上方切换 — **强随机**(全随机字符,适合账户密码/加密密钥)、**可发音**(易拼读,适合设备密码)、**密码短语**(Diceware,适合主密码/恢复短语)。
2. **配置规则**:调整长度滑块(强随机 4-128 位 / 密码短语 4-7 词),勾选字符类型(大小写/数字/符号),需要时可勾选"排除易混淆字符"。
3. **复制或导出**:单击某行复制单条密码,或用 **Copy all** 全复制,或导出 TXT / CSV / JSON 文件。

## 三种模式

- **强随机(默认)**:每个字符独立均匀采样自字符池。默认 26+26+10+32=94 字符,每位贡献 log₂(94) ≈ 6.55 bits;16 位密码约 104 bits,基本无法暴力破解。缺点是不易记忆。
- **可发音**:辅音-元音交替(例 `ralopuvi`、`temukodu`)。每位约 3 bits,24 位密码可达 72 bits,适合手机输入。
- **密码短语 / Diceware**:从 EFF 词表(7776 个词)中随机抽取 4-7 词,每个词 12.9 bits;4 词 ≈ 51.7 bits,5 词 ≈ 64.6 bits,6 词 ≈ 77.5 bits,7 词 ≈ 90.4 bits。XKCD #936 的 *correct horse battery staple* 即此理念。NIST 现推荐长密码短语优于短复杂密码。

## 自定义选项

- **字符类**:大写 A-Z、小写 a-z、数字 0-9、特殊符号(32 个)。可独立开关。
- **排除易混淆字符**:去掉 `0`、`O`、`Q`、`D`、`1`、`l`、`I`、`|`,适合需要口播或手抄的密码(如 WiFi)。
- **排除歧义符号**:去掉 `{`、`}`、`[`、`]`、`(`、`)`、`/`、`\`、`` ` ``、`~`、`'`、`"`、`,`、`;`、`:`、`.`,适合写入配置文件或环境变量。
- **自定义符号集**:粘贴你想用的符号子集,生成器只从你的集合里抽。
- **批量生成**:单次最多 500 条,适合测试或团队开通账号。
- **导出格式**:TXT / CSV / JSON,文件直接下载到本地。

## 熵值与密码强度

密码的**熵**(entropy)以 bits 衡量,N bits 熵的密码平均需要 **2^(N-1) 次**尝试才能暴力破解:

| 熵(bit) | 尝试次数 | 现实破解时间(1 万亿次/秒) |
|---|---|---|
| 28 | 1.3 × 10⁸ | < 1 秒 |
| 40 | 5.5 × 10¹¹ | ~ 9 分钟 |
| 60 | 5.8 × 10¹⁷ | ~ 7 天 |
| 80 | 6.0 × 10²³ | ~ 19,000 年 |
| 100 | 6.3 × 10²⁹ | ~ 200 亿年 |
| 128 | 1.7 × 10³⁸ | 比宇宙年龄还长 |

可在我们的[密码强度可视化工具](/tools/password-strength/)实时查看任意密码的熵值。

**NIST SP 800-63B 推荐**:用户密码最少 8 位;随机生成密码至少 64 bits 熵(随机 ASCII 至少 10 位,高价值账户 16+ 位);不再强制周期性更换密码;允许全部可打印 ASCII。

## 安全最佳实践

- **不要复用密码**:一次泄露,所有账户沦陷。
- **不要写在小纸条上**:用专业密码管理器(1Password / Bitwarden / KeePass)+ 主密码。
- **不要相信服务端生成器**:打开 DevTools → Network,确认点击 Generate 时**零请求**。
- **配合两步验证**:100 bit 密码若手机被偷也没用,务必启用 TOTP(Authy / Google Authenticator)或硬件密钥(YubiKey)。

## 常见问题

### 什么类型的密码最强?

**长度 + 均匀性。** 16 位随机 ASCII 密码(约 104 bits 熵)指数级强于 8 位"复杂"密码(约 30 bits)。每多一位,暴力破解难度乘 ~94。若需要可记忆,5-6 词的 Diceware 密码短语(60-80 bits)是最强的人机友好选择。

### 什么是好的密码生成器?

好的密码生成器应:(1)**100% 客户端运行**,密码不触网;(2) 使用加密级 RNG(`crypto.getRandomValues()`);(3) 可调字符池与排除规则;(4) 支持批量输出;(5) 算法开源可审计。本工具全部满足,且无广告、无追踪、不记录任何东西。

### 密码生成器是怎么工作的?

现代密码生成器从加密级随机数生成器(CSPRNG)取随机性 — 浏览器即 `crypto.getRandomValues()`。每个字符独立均匀采样自所选字符池。本工具还提供可发音模式(CV 音节)和密码短语模式(EFF Diceware 词表 7776 词)。

### 在线密码生成器安全吗?

**仅当真正客户端运行时。** 打开 DevTools → Sources,生成期间应无 `fetch` / `XHR` 请求。本工具 100% 在浏览器内运行,密码生成于 `crypto.getRandomValues()`,从不连接任何服务端点。**自己验证**比任何隐私声明都可靠。

### 密码和密码短语有什么区别?

**密码**是从字符池抽的随机字符串(如 `T7$mK9pQ2vX`)。**密码短语**是多个随机单词用空格/符号分隔(如 `quartz-monorail-cello-fenugreek`)。密码短语在同等可记忆性下熵高得多 — EFF 4 词短语 ≈ 51 bits,远强于普通 12 位混合密码,且更容易记住。

### 密码应该多长?

**NIST SP 800-63B**(2024 修订)推荐:**最少 8 位**(用户自选)或 **64 bits 熵**(随机生成);高价值账户(银行、主邮箱、密码管理器主密码)**16+ 位随机**或 **5+ 词短语**;加密密钥 / 主密码用 **20+ 位随机**或 **7+ 词短语**。永远多比少好,目标 80 bits 熵以上。

### 熵是什么意思?

熵是**不可预测性**的度量,以 bits 表示。N bits 熵的密码平均需要 2^(N-1) 次猜测才能破解 — 40 bits 密码约 5500 亿次(消费级 GPU 几小时),80 bits 密码约 6 × 10²³ 次(超出任何可预见的算力)。实时查看任意密码熵值,请用我们的[密码强度可视化工具](/tools/password-strength/)。
